The adjacvency-list representation of a graph G=(V,E)G = (V,E)G=(V,E) consists of an array Adj , one for each vertex in VVV. For each u∈Vu\in Vu∈V, the adjacency list Adj[u]Adj[u]Adj[u] contains all the vertices ν\nuν such that (u,ν)∈E(u,\nu)\in E(u,ν)∈E.

  • Use space complexity O(∣V∣+∣E∣)O(|V|+|E|)O(∣V∣+∣E∣)

The adjacency-matrix representation of a graph G=(V,E)G = (V,E)G=(V,E) consists of a matrix AdjAdjAdj such that Adj[u][v]Adj[u][v]Adj[u][v] is 1 if (u,v)∈E(u,v)\in E(u,v)∈E and 0 otherwise where this such matrix is ∣V∣×∣V∣|V|\times|V|∣V∣×∣V∣.

  • Use space complexity O(∣V∣2)O(|V|^2)O(∣V∣2)

We always prefer the adjacency-list representation of a graph because it is more space-efficient than the adjacency-matrix representation and get more robustness in which we can augment on the node of the linked list so that we can present weight graph.

Breadth-First Search​

For a graph G=(V,E)G = (V,E)G=(V,E) and a start vertex s∈Vs\in Vs∈V, the breadth-first search of GGG from sss is a traversal of the graph that visits each vertex u∈Vu\in Vu∈V exactly once and for each edge (u,v)∈E(u,v)\in E(u,v)∈E exactly once (may not traversal all). Such traversal graph is called a breadth-first tree. Moreover, we do color each vertex u∈Vu\in Vu∈V as white, gray, or black. We start with sss as gray and all other vertices as white. We then visit each vertex u∈Vu\in Vu∈V exactly once and for each edge (u,v)∈E(u,v)\in E(u,v)∈E exactly once. Once all adjacent vertices of uuu are visited, we color uuu as black. We do this by maintaining a queue QQQ of vertices that we have discovered but not yet visited. Initially, QQQ contains only sss.

The total run-time of BFS is O(∣V∣+∣E∣)O(|V|+|E|)O(∣V∣+∣E∣) where the initialization takes O(∣V∣)O(|V|)O(∣V∣) time and scanning all the edges takes O(∣E∣)O(|E|)O(∣E∣) time.

Some problems can be solved by BFS: shortest path, connected components, bipartite, two-colorability, etc.

  • shortest path: we do BFS and return the degree of the target node. If we can successfully reach the target node, we will update the degree of the target node. Otherwise, we will see its degree is infinity which is the default value.

For a graph G=(V,E)G = (V,E)G=(V,E) and a start vertex s∈Vs\in Vs∈V, we define predecessor subgraph of GGG from sss where Gs=(Vs,Es)G_s = (V_s,E_s)Gs​=(Vs​,Es​) such that Vs={v∈V∣v.π≠NIL}∪{s}V_s = \{v\in V|v.\pi \ne \text{NIL}\} \cup \{s\}Vs​={v∈V∣v.π=NIL}∪{s} and Es={(v.π,v)∣v∈Vs−{s}}E_s = \{(v.\pi,v)|v\in V_s - \{s\}\}Es​={(v.π,v)∣v∈Vs​−{s}}. Such subgraph is breath-first tree if VsV_sVs​ contains all reachable vertices from sss and ∀v∈Vs\forall v\in V_s∀v∈Vs​ there is a unique path from sss to vvv in GsG_sGs​ (also is the shortest path from sss to vvv).

The BFS algorithm take O(∣V∣+∣E∣)O(|V|+|E|)O(∣V∣+∣E∣) time.

The predecessor subgraph of a DFS froms a depth-first forest comprising several depth-first trees. We also do timestamps for each vertex which notice the arrival time and finished time. (u.d denote discovered time which start from parent time + 1, u.f denote finished time which end with the last child time + 1). Topological Sort​

A topological sort of a directed acyclic graph G=(V,E)G = (V,E)G=(V,E) is a linear ordering of all its vertices such that if (u,v)∈E(u,v)\in E(u,v)∈E, then uuu appears before vvv in the ordering. We can use DFS to find a topological sort of a directed acyclic graph G=(V,E)G = (V,E)G=(V,E). Steps with precondition DAG G=(V,E)G = (V,E)G=(V,E):

  1. Call DFS(G) to compute finishing times v.fv.fv.f for each vertex vvv.
  2. as each vertex is finished, insert it onto the front of a linked list.
  3. return the linked list of vertices.

A directed graph GGG is acyclic   ⟺  \iff⟺ a depth-first search of GGG never produces a back edge.

The topological sort algorithm take O(∣V∣+∣E∣)O(|V|+|E|)O(∣V∣+∣E∣) time.

Using Edge and Transpose Edge, we can find the strongly connected components of a directed graph G=(V,E)G = (V,E)G=(V,E).

  1. Call DFS(G) to compute finishing times v.fv.fv.f for each vertex vvv.
