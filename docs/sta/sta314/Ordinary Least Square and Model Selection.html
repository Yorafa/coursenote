Ordinary Least Square and Model Selection

Ordinary Least Squares(OLS) is a approach to find f^\hat{f}f^​ on linear model where f^(X)=β^X\hat f(X) = \hat\beta Xf^​(X)=β^​X. That is, the things we actually to do is to find β^\hat\betaβ^​ where β^=arg⁡min⁡α∈Rp+11n∑i=1n(yi−xiTα)2=arg⁡min⁡α∈Rp+11n∑i=1n∣∣yi−Xα∣∣22\hat \beta = \arg\min\limits_{\alpha\in \R^{p+1}} \frac{1}{n}\sum_{i=1}^n (y_i - x_i^T \alpha)^2 = \arg\min\limits_{\alpha\in \R^{p+1}} \frac{1}{n}\sum_{i=1}^n ||y_i - X \alpha||^2_2β^​=argα∈Rp+1min​n1​∑i=1n​(yi​−xiT​α)2=argα∈Rp+1min​n1​∑i=1n​∣∣yi​−Xα∣∣22​

  • The Loss function for OLS is L(β,Dtrain)=RSS=∣∣y−Xβ∣∣22L(\beta,D_{train}) = RSS = ||y-X\beta||^2_2L(β,Dtrain​)=RSS=∣∣y−Xβ∣∣22​ , Penality is Pen(β)=0Pen(\beta) = 0Pen(β)=0
  • β^=(XTX)−1XTy\hat \beta = (X^TX)^{-1}X^Tyβ^​=(XTX)−1XTy is the solution of OLS where XXX is full column rank.
  • OLS with larger variance but no bias for the model which is truely linear

After we get β^\hat\betaβ^​ , we first assume each error terms are linear independent with E[ϵ]=0,Var(ϵi)=σ2\mathbb{E}[\epsilon] = 0, Var(\epsilon_i) = \sigma^2E[ϵ]=0,Var(ϵi​)=σ2 and then check:

  • Unbiasedness: E[β^]=β\mathbb{E}[\hat \beta] = \betaE[β^​]=β
  • The covariance matrix of β^\hat \betaβ^​ is: Cov(β^)=σ2(XTX)−1Cov(\hat \beta) = \sigma^2(X^TX)^{-1}Cov(β^​)=σ2(XTX)−1
  • The above two properties imply the l2l_2l2​ estimation error E[∣∣β^−β∣∣22]=σ2Tr[(XTX)−1]\mathbb{E} [||\hat \beta - \beta||_2^2] = \sigma^2Tr[(X^TX)^{-1}]E[∣∣β^​−β∣∣22​]=σ2Tr[(XTX)−1] where XTX=nIp+1X^TX = nI_{p+1}XTX=nIp+1​ so that we have final answer E[∣∣β^−β∣∣22]=σ2(p+1)n\mathbb{E} [||\hat \beta - \beta||_2^2] = \frac{\sigma^2(p+1)}{n}E[∣∣β^​−β∣∣22​]=nσ2(p+1)​.
  • The MSEMSEMSE of estimating β\betaβ increases as ppp gets larger which show the larger variance.
  • If p>np>np>n, then OLS estimator is not unique and its variance is infinite.

To reduce our work to find a better model and avoid p>np> np>n situation, we always want to remove all the predictors without significances. That is, we always process model selection to get best training on linear model. Or we can use Lasso, Ridge or elastic net regression to shrinks the coefficient towards zero which the similar concept as remove those coefficient.

Model Selection​

The way we always use is Subset Selection. We identify a subset of the ppp predictors that we believe to be related to the response.

Best Subset Selection (totally compare 2p2^p2p models):

  1. let M0M_0M0​ be the null model (no predictors   ⟹  M0=β^0\implies M_0 = \hat\beta_0⟹M0​=β^​0​ which is the sample mean)
  2. For Mi,i=1,2,…,pM_i, i = 1,2,\ldots,pMi​,i=1,2,…,p (here the ppp states the max features we have):
    1. Fit all (pi){p\choose i}(ip​) models, then we have (pi)Mi{p\choose i} M_i(ip​)Mi​ models
    2. Pick the one with smallest RSS/ largest R2R^2R2, fix that one as true MiM_iMi​

Stepwise Subset Selection:

Forward Stepwise (totally compare 1+p(p+1)21 + \frac{p(p+1)}{2}1+2p(p+1)​ models):

  1. let M0M_0M0​ be the null model
  2. For Mi,i=0,1,2,…,p−1M_i, i = 0, 1,2,\ldots,p - 1Mi​,i=0,1,2,…,p−1
    1. Consider all p−ip-ip−i models that augment the predictors in MiM_iMi​ with one additional predictor
    2. Pick the one with smallest RSS/ largest R2R^2R2, fix that one as Mi+1M_{i + 1}Mi+1​

Backward Stepwise (totally compare 1+p(p+1)21 + \frac{p(p+1)}{2}1+2p(p+1)​ models):

  1. let MpM_pMp​ be the full model
  2. For Mi,i=p,p−1,…,1M_i, i = p, p - 1, \ldots, 1Mi​,i=p,p−1,…,1
    1. Consider all iii models that contain all but one of the predictors in MiM_iMi​ for total i−1i-1i−1 predictors.
    2. Pick the one with smallest RSS/ largest R2R^2R2, fix that one as Mi−1M_{i-1}Mi−1​

After subset selection, we have many training model. If we have such test set, we can do test MSE for each model and select the one with low test error. If not, we can do the method without test data where using cross-validated prediction error, CpC_pCp​, AICAICAIC, BICBICBIC, or adjusted R2R^2R2

Cons and Pros​

Best Subset Selection:

  • can be used for other types of models (i.e. logistic regression)
  • can't be used for n<pn < pn<p
  • much computation

Forward Stepwise Selection:

  • less computation
  • can be used in high-dimensional setting with n<pn<pn<p
  • may not find the best possible since greedy

Backward Stepwise Selection:

  • less computation
  • only used in n>pn>pn>p
  • may not find the best possible since greedy
